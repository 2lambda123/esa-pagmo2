Miscellanea¶

class pygmo.thread_safety¶

Thread safety level.

This enum defines a set of values that can be used to specify the thread safety of problems, algorithms, etc.

basic = pygmo.core._thread_safety.basic¶

concurrent operations on distinct objects are safe

Type

Basic thread safety

constant = pygmo.core._thread_safety.constant¶

constant (i.e., read-only) concurrent operations on the same object are safe

Type

Constant thread safety

none = pygmo.core._thread_safety.none¶

concurrent operations on distinct objects are unsafe

Type

No thread safety

class pygmo.evolve_status¶

Evolution status.

This enumeration contains status flags used to represent the current status of asynchronous evolution/optimisation in pygmo.island and pygmo.archipelago.

See also

pygmo.island.status and pygmo.archipelago.status.

idle = pygmo.core._evolve_status.idle¶

no asynchronous operations are ongoing, and no error was generated by an asynchronous operation in the past (value = 0)

Type

Idle

busy = pygmo.core._evolve_status.busy¶

asynchronous operations are ongoing, and no error was generated by an asynchronous operation in the past (value = 1)

Type

Busy

idle_error = pygmo.core._evolve_status.idle_error¶

no asynchronous operations are ongoing, but an error was generated by an asynchronous operation in the past (value = 2)

Type

Idle with error

busy_error = pygmo.core._evolve_status.busy_error¶

asynchronous operations are ongoing, and an error was generated by an asynchronous operation in the past (value = 3)

Type

Busy with error

pygmo.set_serialization_backend()¶

Set pygmo’s serialization backend.

This function allows to specify the serialization backend that is used internally by pygmo for the (de)serialization of pythonic user-defined entities (e.g., user-defined pythonic problems, algorithms, etc.).

By default, pygmo uses the cloudpickle module, which extends the capabilities of the standard pickle module with support for lambdas, functions and classes defined interactively in the __main__ module, etc.

In some specific cases, however, different serialization backends might work better than cloudpickle, and thus pygmo provides the possibility for the cognizant user to switch to another serialization backend.

The valid backends are:

  • 'pickle' (i.e., the standard Python pickle module),

  • 'cloudpickle',

  • 'dill' (from the dill library).

Warning

Setting the serialization backend is not thread-safe: do not set the serialization backend while concurrently setting/getting it from another thread.

Parameters

name – the name of the desired backend

Type name

str

Raises TypeError

if name is not a str

Raises ValueError

if name is not one of ['pickle', 'cloudpickle', 'dill']

Raises ImportError

if name is 'dill' but the dill module is not installed

pygmo.get_serialization_backend()¶

Get pygmo’s serialization backend.

This function will return pygmo’s current serialization backend (see set_serialization_backend() for an explanation of the available backends).

Returns

the current serialization backend (as a Python module)

Rtype

types.ModuleType

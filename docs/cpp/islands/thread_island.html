#include <pagmo/islands/thread_island.hpp>

class thread_island¶

Thread island.

This class is a user-defined island (UDI) that will run evolutions directly inside the separate thread of execution within pagmo::island.

thread_island is the UDI type automatically selected by the constructors of pagmo::island on non-POSIX platforms or when both the island’s problem and algorithm provide at least the pagmo::thread_safety::basic thread safety guarantee.

void run_evolve(island &isl) const¶

Run evolve.

This method will use copies of isl’s algorithm and population, obtained via island::get_algorithm() and island::get_population(), to evolve the input island’s population. The evolved population will be assigned to isl using island::set_population(), and the algorithm used for the evolution will be assigned to isl using island::set_algorithm().

Parameters

  • isl: the pagmo::island that will undergo evolution.

Exceptions

  • std::invalid_argument: if isl’s algorithm or problem do not provide at least the pagmo::thread_safety::basic thread safety guarantee.

  • unspecified: any exception thrown by:

    • island::get_algorithm(), island::get_population(),

    • island::set_algorithm(), island::set_population(),

    • algorithm::evolve().
